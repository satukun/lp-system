# toB LP生成ウィザード

## 概要

toB向けランディングページを「ウィザード形式で入力」→「プレビュー確認」→「エディタで編集」の3ステップで生成できるWebアプリケーション。
ターゲットユーザーは、LP制作ノウハウを持たない営業・マーケティング担当者。

## ドメイン知識層（変更禁止）

このプロジェクトは以下のファイルを**不変のルールブック**として参照する。
Generator・Evaluator はLP関連機能を実装・評価する際に必ずこれらを読むこと。

| ファイル | 役割 |
|---------|------|
| `SKILL.md` | LP生成の全体フロー・品質チェックリスト |
| `references/design-system.md` | カラーパレット・タイポグラフィ・スペーシング等のビジュアルルール |
| `references/lp-structure.md` | S1〜S11 セクション構造・レイアウトパターン定義 |
| `references/content-source-template.md` | コンテンツ原稿テンプレート（[固定]/[可変]の区別） |

## プロジェクト種別

- **フロントエンドのみ**: バックエンドなし。Next.js App Router のクライアントコンポーネントで構成
- **モバイルWeb / PWA**: デバイス切替（PC/Tablet/SP）プレビューあり

## UI/UX 要件

### ページ構成と遷移
```
/ (ウィザード) → /complete (完了確認) → /editor (詳細編集)
                                ↑
                         ウィザードに戻る
```

### デザイントークン（アプリUI用）
- アプリ自体のUIは `src/app/globals.css` の CSS変数（`--col-*`）を使用
- LP プレビュー内は `references/design-system.md` の CSS変数（`--color-*`）を使用
- **この2つは完全に独立しており、混在させてはならない**

### レスポンシブ
- アプリUI: デスクトップ専用（エディタの性質上）
- LPプレビュー内: `/complete` のデバイス切替で確認可能

## 非機能要件

- データ永続化: localStorage のみ（バックエンドなし）
- パフォーマンス: フォーム入力のプレビュー反映はデバウンス400ms
- 画像: placeholder.co を使用（`references/lp-structure.md` の規定に従う）
- 外部依存: CDN禁止（生成HTMLはすべてインライン）

## 制約事項

- `SKILL.md` および `references/` の内容は変更しない
- LP生成ルールはドメイン知識層（references/）が唯一の正解
- バックエンド・DBは実装しない（Sprint 6以降も localStorage ベース）
