# toB LP構造定義（lp-structure.md）

> このファイルはtoB向けランディングページのセクション構造を定義します。
> AIはこの構造に従い、セクションの順序・レイアウト型・必須要素を遵守してHTMLを生成してください。
> コンテンツの実データは `content-source.md` を参照してください。

---

## S1: Header

- **役割**: ナビゲーションとブランド認知
- **レイアウト**: 左ロゴ / 中央メニュー（最大4項目） / 右CTA
- **PC**: 横並び1行、固定ヘッダー（sticky）
- **SP**: ロゴ左 + ハンバーガー右、メニューはドロワー展開
- **コンテンツ参照**: `content-source.md > S1`
- **必須要素**: ロゴ、メニュー項目、CTAボタン

---

## S2: Hero & Trust Badges

- **役割**: ファーストビュー。サービスの価値を一瞬で伝える
- **レイアウト**: 左コピー・CTA / 右ビジュアル（6:4比率）
- **PC**: 2カラム、左テキスト・右画像、min-height: 600px
- **SP**: 1カラム縦積み（コピー → 画像 → CTA）
- **必須要素**:
  - メインコピー（1〜2行、32px以上）
  - サブコピー（補足情報、1〜2行）
  - CTAボタン（プライマリ）
  - 信頼指標バッジ（導入社数・No.1表記など、最大3つ）
- **コンテンツ参照**: `content-source.md > S2`

---

## S3: Message

- **役割**: サービス概要を端的に伝えるブリッジセクション
- **レイアウト**: 中央揃え1カラム
- **PC/SP共通**: max-width: 720px、中央配置
- **必須要素**:
  - セクション見出し（24px以上）
  - 本文（1〜3文、サービスの概要）
- **コンテンツ参照**: `content-source.md > S3`

---

## S4: Problem Identification

- **役割**: ターゲットの課題を提示し共感を得る
- **レイアウト**: 3カラム カード型
- **PC**: 3列均等グリッド（gap: 24px）
- **SP**: 1カラム縦積み
- **カード構成**: アイコン（イラスト指示） / 見出し / 説明文
- **カード数**: 3枚（固定推奨、MVP）
- **必須要素**:
  - セクション見出し（「こんなお悩みありませんか？」等）
  - 課題カード × 3
- **コンテンツ参照**: `content-source.md > S4`

---

## S5: Features & Benefits

- **役割**: サービスの特徴・強みを提示
- **レイアウト**: 3カラム カード型
- **PC**: 3列均等グリッド（gap: 24px）
- **SP**: 1カラム縦積み
- **カード構成**: ポイント番号（POINT1等） / タイトル / 説明文 / アイコンまたは画像指示
- **カード数**: 3枚（固定推奨、MVP）
- **必須要素**:
  - セクション見出し（「選ばれる理由」等）
  - 特徴カード × 3
- **コンテンツ参照**: `content-source.md > S5`

---

## S6: Categories

- **役割**: サービス提供可能な業種・カテゴリを網羅的に見せる
- **レイアウト**: 6グリッド カード型（3列 × 2行）
- **PC**: 3列2行グリッド（gap: 16px）
- **SP**: 2列グリッド
- **カード構成**: 画像指示 / カテゴリ名 / 補足テキスト
- **カード数**: 6枚（固定推奨、MVP）
- **必須要素**:
  - セクション見出し
  - カテゴリカード × 6
  - セクション下部にCTAボタン × 2（プライマリ + セカンダリ）
- **コンテンツ参照**: `content-source.md > S6`

---

## S7: Case Studies

- **役割**: 導入事例で信頼性と具体性を示す
- **レイアウト**: 3カラム カード型 + 下部に導線リンク
- **PC**: 3列均等グリッド（gap: 24px）
- **SP**: 1カラム縦積み
- **カード構成**: 画像指示 / 企業名 / 課題と成果の要約（2〜3行）
- **カード数**: 3枚（固定推奨、MVP）
- **必須要素**:
  - セクション見出し（「導入事例」等）
  - 事例カード × 3
  - 「他の導入事例を見る」リンク
  - セクション下部にCTAボタン × 2
- **コンテンツ参照**: `content-source.md > S7`

---

## S8: Implementation Flow

- **役割**: 契約から運用開始までの流れを可視化
- **レイアウト**: 水平ステップ形式（PC） / 垂直ステップ形式（SP）
- **PC**: 横並びステップ（番号 → タイトル → 説明）、矢印またはラインで接続
- **SP**: 縦並びステップ
- **ステップ数**: 3〜4（固定推奨、MVP）
- **ステップ構成**: ステップ番号 / アイコン指示 / タイトル / 説明文
- **必須要素**:
  - セクション見出し（「お申込みの流れ」等）
  - ステップ × 3〜4
- **コンテンツ参照**: `content-source.md > S8`

---

## S9: Lead Form & FAQ

- **役割**: リード獲得フォームとFAQの2カラム構成
- **レイアウト**: 左FAQ / 右フォーム（PC）、縦積み（SP）
- **PC**: 2カラム（5:5 または 6:4）
- **SP**: FAQ → フォームの縦積み

### FAQ部分
- アコーディオン形式（クリックで開閉）
- 質問数: 5問（固定推奨、MVP）

### フォーム部分
- **必須フィールド**: お名前 / 会社名 / メールアドレス / 電話番号 / 従業員数（セレクト） / プライバシーポリシー同意チェック
- 送信ボタン（プライマリCTA）
- **コンテンツ参照**: `content-source.md > S9`

---

## S10: Closing Message

- **役割**: 最終CTAへの後押し
- **レイアウト**: 1カラム中央揃え
- **PC/SP共通**: 背景色あり（プライマリカラー系）、中央配置
- **必須要素**:
  - マイクロコピー（「まずは資料請求してみませんか？」等）
  - CTAボタン × 2（プライマリ + セカンダリ）
- **コンテンツ参照**: `content-source.md > S10`

---

## S11: Footer

- **役割**: 企業情報・法的リンク
- **レイアウト**: 左ロゴ / 右リンク群
- **PC**: 横並び
- **SP**: 縦積み中央揃え
- **必須要素**:
  - ロゴ
  - リンク群（アプリについて / サポート / プライバシーポリシー / 利用規約 / 電子公告）
  - コピーライト
- **コンテンツ参照**: `content-source.md > S11`

---

## 全体ルール

- **セクション順序**: S1 → S2 → S3 → S4 → S5 → S6 → S7 → S8 → S9 → S10 → S11（この順序を必ず守ること）
- **レスポンシブ**: PC（1024px以上）/ SP（1023px以下）の2ブレイクポイント
- **CTAの一貫性**: ページ内のプライマリCTAはすべて同じ文言・リンク先にする
- **画像**: placeholder.co を使用し、ビジュアル指示をaltテキストとして埋め込む
- **出力形式**: 単一の index.html ファイル（CSS・JSインライン）
